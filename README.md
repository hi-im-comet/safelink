# SafeLink AI

**우리 집 기준 생활재난 안전 에이전트** — 프론트엔드 데모(MVP).

폭염·홍수/침수·한파·미세먼지·태풍/강풍까지, 오늘의 날씨·특보를 개인의 주거환경·건강·가족 상태와 결합해
*지금 누가 위험하고, 왜 위험하며, 무엇을 해야 하는지*를 쉬운 말로 안내합니다.

- **사용자 앱** — 내 집·가족 정보를 등록하면 재난 유형별 위험도와 오늘 할 일을 알려줍니다.
- **관리자 대시보드** — 지자체·복지관·생활지원사가 우선 확인 대상과 대응 현황을 관리합니다.

> 위험도 판단은 **규칙 기반 코드**가 담당합니다(화면에서는 "AI 자동분류"로 표기). LLM은 긴급 판단이 아니라 보호자 공유 문구·전화 스크립트·체크리스트 문장 생성 등 **표현**에만 선택적으로 사용하며, 개인정보는 LLM에 직접 전달하지 않는 구조입니다.
> **별도 회원가입·로그인 없이 누구나 바로 체험**할 수 있고, 외부 API 키나 AI API 키 없이 전체 데모가 동작합니다. 사용자 입력 정보는 브라우저 `localStorage`에만 저장되며 서버로 전송되지 않습니다.

> 본 서비스는 공식 재난안전 안내를 보조하는 데모이며, 긴급 상황에서는 119 및 지자체 안내를 우선합니다.

## 데모 운영 방식

현재 버전은 대회 제출용 MVP로, 별도 로그인 없이 누구나 바로 체험할 수 있도록 구성했습니다. 관리자 화면은 서비스 흐름을 보여주기 위한 시연용 취약가구 데이터를 사용하며, 사용자 앱은 입력한 정보를 브라우저 localStorage에 저장해 개인 맞춤형 위험도와 행동요령을 제공합니다. 실제 서비스 확장 시에는 지자체 계정 인증, 보호자 계정, 취약가구 DB, 공공데이터 API 연동이 가능합니다.

## 심사위원 체험 시나리오

1. 홈에서 **관리자 대시보드** 진입
2. 재난 유형을 **폭염 / 홍수·침수 / 한파 / 미세먼지 / 태풍·강풍**으로 전환
3. **우선 확인 대상**과 **필요한 조치** 확인
4. 홈으로 돌아가 **사용자 앱** 진입
5. **본인 안전 모드** 또는 **보호자 확인 모드** 선택
6. 정보 입력 후 **위험도와 행동요령** 확인
7. **도움 요청** 페이지에서 요청 문구 자동 생성 확인

> 회원가입·로그인 없이 위 7단계를 그대로 따라가면 핵심 기능을 모두 확인할 수 있습니다.

## 실행 방법

```bash
npm install
npm run dev
```

접속: [http://localhost:3100](http://localhost:3100/)

빌드 / 프로덕션 실행:

```bash
npm run build
npm start
```

> Node 18.18+ / Next.js 15 / React 19 / TypeScript / Tailwind CSS.

## 배포 (Vercel)

로그인·DB·환경변수가 없어 추가 설정 없이 배포됩니다.

1. GitHub 저장소를 [Vercel](https://vercel.com/new)에 임포트
2. Framework Preset은 **Next.js** 자동 감지 — 별도 설정 불필요
3. Deploy

또는 CLI:

```bash
npm i -g vercel
vercel        # 프리뷰 배포
vercel --prod # 프로덕션 배포
```

## 화면

| 경로 | 설명 |
|------|------|
| `/` | 랜딩 — 오늘 활성 위험(5종) + 관리자 / 사용자 앱 / 서비스 소개 진입 |
| `/admin` | **관리자 대시보드(데스크톱)** — 재난 유형 칩으로 리스트·위험도 전환, 우선순위 리스트(위험 점수·취약 조건·필요한 조치·담당자·상태·오늘 처리), 오늘 운영 브리핑 |
| `/user` | **사용자 앱(모바일)** — 첫 진입 시 **본인 안전 / 보호자 확인** 모드 선택 → 정보 입력 → 모드별 홈(재난 유형별 위험도·주의 단계·태그·설명·체크리스트·도움 요청·대피소·공유 문구) |
| `/user/profile` | 정보 수정 — 모드별 입력 폼(섹션 분리) + **사용 모드 변경**·데모 초기화 |
| `/user/help` | 도움 요청 — 긴급 안내·현재 위험 요약·요청 유형 선택·모드별 요청 문구 자동 생성 → 관리자 리스트로 전달(도움요청 최상단 고정) |
| `/about` | 서비스 소개 — 작동 흐름, AI 사용 방식, 대응 재난, 확장 가능성 |

> 사용자 앱은 **본인 안전 모드**(내가 쓰는 앱)와 **보호자 확인 모드**(가족·부모님을 대신 확인)로 나뉘며, 문구·버튼·요청/공유 문구가 모드에 맞게 바뀝니다. 모드는 `localStorage`에 저장되고 "사용 모드 변경"으로 다시 고를 수 있습니다.
> **재난 유형 선택**(폭염·홍수/침수·한파·미세먼지·태풍/강풍)에 따라 관리자·사용자 화면의 카드·위험도·문구·체크리스트가 함께 바뀝니다. 선택 상태는 두 화면이 공유합니다.
> 사용자가 도움 요청을 보내면 관리자 우선순위 리스트 최상단에 실시간으로 반영됩니다.
> 데모를 처음 상태로 되돌리려면 사용자 앱의 **데모 초기화** 또는 관리자 상단의 **데모 초기화** 버튼을 누르세요(localStorage 삭제).

## 위험도 규칙 (코드 계산)

```
오늘 위험도 = 재난 강도 40% + 개인·주거 취약도 60%
등급: 80↑ 위험 높음 · 60~79 주의 · 40~59 행동요령 · 40↓ 안정
도움 요청 가구는 점수와 별개로 관리자 우선순위 최상단에 고정됩니다.
```

규칙 엔진은 `lib/hazards.ts`의 `assessRisk()`에 있으며 LLM을 호출하지 않습니다.

## 구조

```
app/                    라우트 (랜딩 / admin / user / about)
components/             UI (재난 칩·우선순위 행·모바일 다이얼·입력 폼 …)
lib/
  types.ts              공통 타입 (Hazard / Household / UserProfile …)
  hazards.ts            재난 5종 정의 + 규칙 기반 위험 분석 엔진 + 보호자 문구
  profile.ts            사용자 입력 폼 구성·기본값
  risk/score.ts         관리자 우선순위·통계 헬퍼
  mock/households.ts    시연용 가구 데이터 (재난 유형별)
  store/AppState.tsx    공유 상태 (프로필·재난 선택·도움 요청, localStorage)
  ui/grade.ts           등급 색 클래스
```

## 디자인

연두·초록·민트가 dominant인 카드형 UI. 재난 유형별로 배지 색감만 살짝 다르게(폭염 warm, 홍수 blue, 한파 teal, 미세먼지 clay, 태풍 slate) 구분합니다.
히어로/배경은 사진 없이 CSS·SVG·gradient로 구성했고, 토큰은 `tailwind.config.ts`와 `app/globals.css`에 정리되어 있습니다.

> 향후 공공데이터 API, 기상특보 API, 지자체 대피소 데이터와 연동해 실제 지역 기반 재난 대응 서비스로 확장할 수 있습니다.
